Traefik Explanation
===================

1. What are the advantages of using Traefik with Docker?
--------------------------------------------------------
- Automatic service discovery via the Docker socket — no manual upstream configuration.
- Dynamic routing: add/remove containers and Traefik updates routes without restart.
- Built-in dashboard and API for monitoring routers, services, and middlewares.
- Native support for Let's Encrypt, TLS, and middleware (auth, rate limiting, etc.).
- Label-based configuration keeps routing rules next to each service in docker-compose.


2. How does Traefik automatically detect routing?
-------------------------------------------------
Traefik watches the Docker socket for container start/stop events. When a container has `traefik.enable=true` and routing labels (e.g. Host rule, entrypoints), Traefik registers it as a backend and creates a router. No static config file edits are needed per service.


3. What is the difference between Traefik and Nginx/HAProxy?
------------------------------------------------------------
Traefik is cloud-native and dynamic: configuration comes from Docker labels or orchestrator metadata, ideal for microservices that scale up and down.

Nginx and HAProxy typically use static config files; changes require editing config and reloading. They are mature, high-performance, and better when topology is stable.

Traefik prioritizes automation and integration with containers/Kubernetes; Nginx/HAProxy prioritize manual control, fine-tuning, and proven production deployments.
